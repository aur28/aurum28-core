# Contributing to Aurum28 Core

Thank you for your interest in contributing to Aurum28.

## Guidelines

- All contributions must be reviewed before merging.
- Ensure code is secure, clean, and well documented.
- Do not include private keys, secrets, or sensitive data.

## Reporting Issues

Please use the GitHub Issues section to report bugs or suggestions.

## Development Philosophy

Aurum28 prioritizes:

- Security
- Simplicity
- Transparency
- Long-term stability
